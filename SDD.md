## 1. 文件目的

本文件定義「員工投票系統」之軟體設計規格，作為需求確認、系統設計、開發拆分、測試驗收與後續維護之依據。

**本系統目標：**

1. 提供員工以工號、生日 8 碼、驗證碼登入後進行投票。
2. 若員工有多個有效活動，同一頁面顯示所有活動及其候選人，統一送出；若僅一個，登入後直接進入該活動同頁面。
3. 投票完成後回到活動列表頁，顯示各活動即時得票佔比。
4. 若員工所有活動均已投票完畢（含登入時即已全部投過），直接進入列表頁顯示結果。
5. 提供管理後台建立投票活動（含說明欄位）、匯入投票名單、建立候選人、查看開票結果。
6. 前台採純 HTML 頁面，使用 JavaScript 呼叫 API 並渲染內容。
7. 後端以 .NET 8 / C# 建構，資料庫使用 SQLite。
8. 以前端 Cookie 儲存 session 資訊，各頁面先以 Cookie 檢查登入狀態，再由後端做實際驗證。

## 3. 名詞定義

| 名詞 | 說明 |
|------|------|
| 投票活動 | 一次完整投票事件，具有名稱、說明、時間範圍、活動代號 |
| 候選人 | 供員工投票選擇的對象 |
| 可投票名單 | 有資格投票的員工清單 |
| 已投票 | 同一活動中該員工已完成投票，不可再次投票 |
| 活動代號 | 系統產生之唯一識別碼 |
| 驗證碼 | 登入頁顯示之圖形或文字驗證碼，用於阻擋機器請求 |
| 有效活動 | 目前時間落在活動起訖範圍內，且該員工在可投票名單中的活動 |
| 批次投票送出 | 員工在單一操作中同時送出多個活動的投票選擇 |
| 得票佔比 | 各候選人得票數佔該活動總有效票數之百分比，以橫條圖呈現 |
| 結果查閱設定 | 控制員工是否可查看已投票活動結果的開關與時間範圍（v0.4 新增） |
| 可查閱活動 | 已投票、開啟結果查閱、且在查閱時間範圍內的活動，供員工查看得票佔比（v0.4 新增） |

## 4. 系統角色

### 4.1 員工使用者

**可執行：**
- 進入登入頁
- 使用工號、生日、驗證碼登入
- 於同一頁面查看所有有效活動並選擇候選人
- 一次送出所有活動的投票（統一確認彈窗）
- 查看各活動得票佔比（投票完成後）

**不可執行：**
- 查看完整票數或候選人排名
- 修改活動或候選人
- 匯入名單或修改管理資料

### 4.2 管理者

**可執行：**
- 登入後台
- 建立／修改投票活動（含說明欄位）
- 設定投票起訖時間
- 匯入可投票名單
- 建立／修改候選人資訊
- 查看投票統計與得票圖（於活動詳情頁籤內）
- 管理管理者帳號

## 5. 使用情境總覽

### 5.1 員工登入與頁面導流流程（v0.4 更新）

1. 使用者進入登入頁。
2. 輸入工號、生日 8 碼（yyyymmdd）、驗證碼。
3. 系統驗證登入資料。
4. 後端查詢該工號於目前有效期間內且名單匹配的活動。
5. 後端同時查詢該工號已投票且在「結果查閱期限」內的活動（v0.4 新增）。
6. 若無符合活動（無有效活動且無可查閱活動），登入失敗並提示無可參與活動。
7. 若所有有效活動皆已投票完畢，進入「活動結果列表頁」（顯示有效活動 + 可查閱活動的佔比）。
8. 若有一個以上尚未投票的有效活動，進入「活動投票頁」（所有活動同頁呈現，包含可查閱活動）。

### 5.2 員工投票流程（v0.3 更新）

1. 員工進入活動投票頁，頁面顯示所有有效活動區塊。
2. 每個活動區塊包含：活動名稱、說明、代號、起訖時間，以及候選人卡片清單。
3. 已投票的活動顯示「已投票」狀態與即時得票佔比橫條，不可再選。
4. 未投票的活動顯示候選人卡片，使用者點選卡片進行選取，每活動同時間只能選一人。
5. 頁面底部有固定的「送出所有投票」按鈕（sticky footer）。
6. 按鈕狀態規則：
   - 所有未投票活動皆未選候選人 → 按鈕 disabled
   - 部分活動已選候選人 → 按鈕可按，底部提示尚未選擇的活動名稱
   - 所有未投票活動皆已選候選人 → 按鈕可按，無額外提示
7. 按下按鈕後，彈出確認視窗，列出「活動名稱 → 候選人姓名」清單。
8. 使用者確認後，後端批次寫入所有投票紀錄。
9. 完成後頁面更新：已投票活動顯示佔比；若所有活動皆已完成，進入結果列表狀態。

### 5.3 已完成所有投票的結果列表頁（v0.4 更新）

1. 適用情境：員工登入時所有有效活動均已投票，或投票後所有活動均完成。
2. 頁面顯示「感謝您完成所有投票」banner。
3. 每個活動區塊顯示活動資訊、已投票標記，以及即時得票佔比橫條圖。
4. 同時顯示「可查閱」的已投票活動（含已結束活動），顯示「已結束」badge（v0.4 新增）。
5. 不顯示候選人選擇卡片，不顯示送出按鈕。

### 5.4 結果查閱設定說明（v0.4 新增）

1. 每個投票活動包含以下結果查閱相關欄位：
   - **是否開放查閱**（IsResultViewable）：布林值，控制員工是否可查看結果
   - **查閱開始時間**（ResultViewStartTime）：員工可開始查看結果的時間
   - **查閱結束時間**（ResultViewEndTime）：員工可查看結果的截止時間
2. 若「是否開放查閱」為 false，員工無法查看該活動的得票佔比，不論時間設定為何。
3. 若「是否開放查閱」為 true，員工須在「查閱開始時間」至「查閱結束時間」範圍內才能查看結果。
4. 查閱時間範圍可設定為投票期間內（即時查看）或投票結束後（延後公布）。
5. 結果查閱設定僅影響員工前台的可見性，不影響後台管理者查看開票結果。
6. 員工必須「已投票」才能查看該活動的結果，未投票的員工即使在查閱時間範圍內也無法看到結果。

### 5.5 管理者建立活動流程（v0.4 更新）

1. 管理者登入後台。
2. 點擊「新增活動」按鈕，彈出活動編輯彈窗（Modal）。
3. 輸入活動名稱、說明（選填）、開始時間、結束時間。
4. 設定結果查閱：是否開放查閱（預設關閉）、查閱開始時間、查閱結束時間。
5. 系統產生活動代號。
6. 儲存後關閉彈窗，列表自動刷新。

### 5.6 管理者編輯活動流程（v0.4 新增）

1. 管理者在活動列表點擊「查看詳情」按鈕。
2. 彈出活動編輯彈窗（Modal），與新增活動共用相同設計。
3. 彈窗包含多個頁籤：基本資料、可投票名單、候選人管理、開票頁面。
4. 修改完成後點擊「儲存變更」，關閉彈窗並刷新列表。
5. 點擊「取消」或關閉按鈕，捨棄變更並關閉彈窗。

### 5.7 匯入可投票名單流程

1. 管理者在活動編輯彈窗中切換至「可投票名單」頁籤。
2. 於大型文字框貼上名單，每行格式：`工號 空格 生日`（例：`1233C00 1897/01/02`）。
3. 系統解析、驗證格式、寫入名單。
4. 顯示匯入成功筆數、失敗筆數、錯誤明細。

## 6. 功能需求

### 6.1 前台登入頁

#### 6.1.1 欄位
- 工號
- 生日 8 碼（yyyymmdd）
- 驗證碼輸入框
- 驗證碼圖片或文字
- 登入按鈕

#### 6.1.2 驗證規則
- 工號：必填，不可空白，最大長度建議 20
- 生日：必填，固定 8 碼，只允許數字，格式 `yyyymmdd`
- 驗證碼：必填，需與伺服器發送內容一致
- 工號 + 生日需存在於至少一個有效活動的可投票名單中

#### 6.1.3 登入後導向規則（v0.3 更新）
- 若員工在所有有效活動中均已投票完畢 → 直接進入活動結果列表頁
- 若有至少一個有效活動尚未投票 → 進入活動投票頁（所有活動同頁）
- 若 0 個有效活動 → 登入失敗，提示目前無可參與投票活動

#### 6.1.4 成功後建立 Cookie
- `employee_session`
- `role=employee`

#### 6.1.5 失敗情境
- 驗證碼錯誤
- 工號不存在
- 工號與生日不匹配
- 不在任何有效活動名單中
- 無有效活動
- session 建立失敗

### 6.2 前台活動投票頁（v0.3 大幅更新）

#### 6.2.1 頁面結構
- 頁面載入時依 Cookie 檢查登入狀態，再由後端 API 驗證 session 合法性
- 頁面以垂直堆疊方式顯示所有有效活動區塊
- 每個活動區塊包含：活動名稱、說明（若有）、活動代號、起訖時間、狀態 badge
- 頁面底部 sticky footer 放置統一送出按鈕

#### 6.2.2 活動區塊顯示規則
- 已投票的活動：顯示「已投票」badge + 即時得票佔比橫條圖
- 未投票的活動：顯示候選人卡片清單（可點選）

#### 6.2.3 候選人卡片規則（與 v0.2 相同）
- 點擊卡片任一區域即視為選取
- 同活動同時間只能選取一位候選人
- 被選取卡片需有明顯外框或高亮狀態（含右上角勾勾圖示）
- 再次點選其他卡片時，前一張取消選取
- 每張卡片包含：圖片、姓名、介紹文字

#### 6.2.4 統一送出按鈕規則（v0.3 新增）
- 固定於頁面底部（sticky footer），寬度跟隨內容區域最大寬度
- 按鈕文字：「送出所有投票」
- 若所有未投票活動皆未選候選人：按鈕 disabled
- 若部分未投票活動已選候選人：按鈕可按，footer 下方顯示尚未選擇的活動名稱提示
- 若所有未投票活動皆已選候選人：按鈕可按，無額外提示
- 若所有活動均已投票：footer 不顯示

#### 6.2.5 確認視窗（v0.3 更新）
- 標題：確認送出
- 內容：列表顯示每筆「活動名稱 → 候選人姓名」
- 備註：「送出後無法修改」
- 按鈕：取消 / 確認送出

#### 6.2.6 投票送出規則
後端必須批次驗證每筆投票：
- Session 合法
- 活動仍在投票期間
- 該使用者在可投票名單內
- 尚未投票
- 候選人屬於此活動且狀態有效
- 候選人圖片存在且有效

所有驗證通過後原子性寫入；其中一筆失敗則整批失敗並回傳詳細錯誤。

### 6.3 前台得票佔比顯示（v0.3 新增）
- 以橫條圖（bar chart）呈現，非圓餅圖

### 6.6 活動詳情彈窗（v0.4 大幅更新）

#### 6.6.1 彈窗設計規範

| 項目 | 說明 |
|------|------|
| 彈窗觸發 | 點擊列表「查看詳情」按鈕，或點擊「新增活動」按鈕 |

## 11. 業務規則（v0.3 更新）

### 11.1 投票規則

1. 同一活動中，一位員工只能投一次。
2. 員工必須存在於該活動可投票名單。
3. 活動必須在有效投票時間內。
4. 候選人必須屬於該活動且為啟用狀態。
5. 候選人必須有圖片。
6. 投票送出後不得修改。
7. 前台不顯示員工具體投給誰。
8. 批次送出時，所有投票必須全部通過驗證後才原子性寫入。（v0.3 新增）

### 11.2 活動規則（v0.4 更新）

1. 活動代號唯一。
2. 開始時間不得晚於結束時間。
3. 活動進行中允許延長截止時間。
4. 活動進行中不得將結束時間縮短到早於當前時間。
5. 活動已結束後不可修改時間。
6. 有投票紀錄後，不可任意刪除活動。
7. 若開放查閱結果，查閱開始時間與結束時間皆為必填（v0.4 新增）。
8. 查閱開始時間必須小於或等於查閱結束時間（v0.4 新增）。

### 11.3 結果查閱規則（v0.4 新增）

1. 「是否開放查閱」決定員工是否可查看該活動的得票佔比。
2. 若開放查閱，員工須在「查閱開始時間」至「查閱結束時間」範圍內才能查看結果。
3. 查閱時間範圍可設定為：
   - 投票期間內（即時查看，例如開始時間 = 投票開始、結束時間 = 投票結束）
   - 投票結束後（延後公布，例如開始時間 = 投票結束、結束時間 = 投票結束後 7 天）
   - 投票期間 + 延長查閱（例如開始時間 = 投票開始、結束時間 = 投票結束後 7 天）
4. 僅「已投票」的員工可在查閱時間範圍內查看該活動的得票佔比。
5. 未投票的員工即使在查閱時間範圍內也無法看到結果。
6. 結果查閱設定不影響後台管理者查看開票結果。
7. 查閱時間範圍可於任何時候修改。

### 11.4 得票佔比顯示規則（v0.3 新增）

1. 前台員工頁：只顯示百分比，不顯示絕對票數。
2. 後台管理頁：顯示絕對票數與百分比。
3. 無投票資料時，前台顯示空狀態，不顯示 0%。

### 11.5 候選人規則

1. 姓名必填。
2. 圖片必填。
3. 同活動下姓名建議不允許重複。
4. 已有投票紀錄之候選人不得物理刪除，改用停用。

### 11.6 管理者規則

1. 密碼需符合複雜度規範。
2. 密碼直接存資料表，不做額外加密。
3. 重設密碼由 IT 直接修改資料表，不提供額外重設流程。

## 12. 非功能需求

### 12.1 安全性
- 前端 Cookie 僅保存必要 session 識別資訊，不可信任其內容作最終授權判斷
- 所有實際權限與投票資格皆由後端再次驗證
- 員工前台與管理後台登入皆必須使用驗證碼
- API 必須防止重複送票
- 批次送出 API 需防止 race condition 造成同一活動重複投票

### 12.2 可用性
- 前台畫面簡潔
- 卡片可於桌面與手機顯示
- 錯誤訊息需明確可理解

### 12.3 一致性
- 批次投票提交需具原子性
- 一次請求中不可對同一活動寫入超過一筆投票

### 12.4 效能
- 單活動候選人數通常不高，SQLite 足以支援
- 匯入名單應支援至少數千筆資料
- 得票佔比計算應在可接受時間內完成

### 12.5 可維護性
- 前後端分離
- 後端分層設計（Presentation / Application / Domain / Infrastructure）
- API 命名一致

## 14. 異常情境設計（v0.3 更新）

### 14.1 員工登入異常
- 驗證碼失效 / 錯誤
- 工號不存在 / 生日不符
- 無符合資格之有效活動

### 14.2 投票異常（v0.3 更新）
- Session 過期
- 重複投票
- 非法 candidateId
- 活動已關閉
- 批次送出中部分活動已被投票（race condition）→ 整批失敗，回傳詳細錯誤
- 後端寫入失敗

### 14.3 匯入異常
- 空白資料
- 格式錯誤
- 重複工號
- 不合法日期

### 14.4 後台異常
- 驗證碼錯誤
- 帳號重複
- 密碼不符複雜度
- 帳號停用
- 無權限操作

## 16. 測試案例摘要（v0.3 更新）

### 16.1 員工登入（v0.4 更新）
- 正確工號 + 正確生日 + 正確驗證碼
- 錯誤生日 / 錯誤驗證碼
- 非名單內工號
- 無有效活動
- 所有活動已投票完畢 → 直接進入結果列表頁
- 部分活動已投票 → 進入投票頁，已投票活動顯示佔比
- 有可查閱活動（已結束但在查閱期限內）→ 登入成功，顯示結果（v0.4 新增）
- 僅有可查閱活動（無有效活動）→ 登入成功，進入結果列表頁（v0.4 新增）

### 16.2 多活動投票（v0.3 新增）
- 多活動同頁顯示正確
- 只選部分活動 → 按鈕可按，提示未選活動
- 全部選齊 → 確認視窗列出所有活動與候選人
- 批次送出成功 → 所有活動顯示已投票 + 佔比
- 批次送出中一筆失敗 → 整批 rollback，顯示錯誤

### 16.3 投票
- 正常完成投票
- 不選候選人直接送出（disabled 狀態，應無法送出）
- 重複投票
- 非法 candidateId
- Session 過期後送出

### 16.4 名單匯入
- 正確資料
- 日期格式混用（yyyy/MM/dd 與 yyyy-MM-dd）
- 重複工號
- 空白行
- 不合法日期

### 16.5 候選人管理
- 上傳合法圖片
- 上傳過大圖片
- 缺少姓名 / 缺少圖片
- 停用已有票數候選人

### 16.6 管理者管理
- 建立符合密碼規則帳號
- 建立不符合密碼規則帳號
- 停用管理者後不可登入

### 16.7 活動列表搜尋與分頁（v0.4 新增）
- 無關鍵字時顯示全部活動（分頁）
- 輸入關鍵字搜尋活動名稱 → 正確篩選
- 輸入關鍵字搜尋活動代號 → 正確篩選
- 切換每頁筆數 → 分頁正確更新
- 搜尋後換頁 → 維持搜尋條件
- 清除搜尋 → 回復完整列表

### 16.8 活動詳情彈窗（v0.4 新增）
- 點擊「新增活動」→ 彈窗開啟，僅顯示基本資料頁籤
- 新增活動成功 → 彈窗關閉，列表刷新
- 點擊「查看詳情」→ 彈窗開啟，顯示所有頁籤
- 編輯基本資料後儲存 → 彈窗關閉，列表刷新
- 點擊取消 → 彈窗關閉，不儲存變更
- 點擊遮罩或 ESC → 彈窗關閉

## 20. 結論

本系統為一套內部用途、快速交付導向的輕量投票系統。v0.4 在 v0.3 基礎上完成以下設計決策：

1. 活動詳情改為彈窗（Modal）形式，與新增活動共用相同 UI，簡化操作流程。
2. 新增「結果查閱期限」欄位，員工可在期限內查看已投票活動的得票佔比。
3. 活動列表新增搜尋與分頁功能，支援大量活動資料的管理需求。
4. 登入邏輯調整，支援顯示可查閱活動（已結束但在查閱期限內的已投票活動）。

此版本已可作為後續拆 API、DB Schema、頁面與開發任務的正式基礎文件。
